# dev-graph 최종 테스트 체크리스트

GOAL.md의 **최종 요구사항(136–148)**과 **최종 성공기준(179–187)**을 직접 손으로 검증하는 절차.
각 항목은 **조작 → 기대 결과**이며 체크박스로 진행 표시. (MVP 흐름 검증은 이 문서로 함께 커버된다.)

> 브랜치: `feat/final-requirements`. 관리자 비밀번호 = `backend/.env`의 `ADMIN_PASSWORD`(기본 `devgraph-admin`).
> 현재 DB에는 이미 **2개 산업**이 시드되어 있다: `튀김유 밸류체인`(치킨/식용유) + `반도체 산업 가치사슬`. 바로 테스트 가능.

---

## 0. 실행

- [ ] DB: `docker compose up -d db`
- [ ] Node 22: `nvm use`
- [ ] 의존성: `pnpm install`
- [ ] 백엔드: `pnpm --filter backend dev` → 로그에 `migrated`/`skip migration` 0001–0012 + `Server is running on port 8080`
- [ ] 프론트: `pnpm --filter frontend dev` → `http://localhost:5173`
- [ ] 헬스: `curl localhost:8080/healthz` → `{"status":"ok"}`

### 실제 LLM(Codex, ChatGPT 구독) 전제
- [ ] `LLM_PROVIDER=codex`(기본), 로컬에 `codex login` 상태(`codex login status` → `Logged in using ChatGPT`)
- [ ] 백엔드를 **codex가 PATH에 있는 셸에서** 실행하거나 `backend/.env`의 `CODEX_BIN`을 codex 절대경로로 지정
- [ ] (API 키 방식으로 바꾸려면 `LLM_PROVIDER=openai` + `OPENAI_API_KEY`)

## A. 공개 산업지도 — 다중 산업 탐색 (최종 #1, 성공기준 1·4)

- [ ] `http://localhost:5173/` → 첫 화면이 **전체 산업지도 그래프**(랜딩/목록 아님)
- [ ] 화면에 **두 산업**이 함께 보인다: 큰 치킨/식용유 밸류체인 + 우측 반도체 가치사슬(약 63노드)
- [ ] 우하단 **미니맵**, 좌하단 줌 컨트롤, 휠 줌/드래그 팬 동작
- [ ] **클러스터 필터**: 상단 `클러스터 전체` 셀렉트 → `반도체 산업 가치사슬` 선택 → 반도체 노드만 선명, 나머지(치킨/식용유) **흐려짐**
- [ ] `튀김유 밸류체인` 선택 → 반대로 흐려짐 / `클러스터 전체`로 복귀
- [ ] 노드 타입 칩(`상품/원재료`·`공정/기능`·`사업자군/산업군`) 토글 → 비해당 흐려짐(숨김 아님)
- [ ] `엣지 타입 필터` 펼쳐 특정 타입만 → 나머지 엣지 흐려짐

## B. 검색 (성공기준 4)

- [ ] 검색창에 `파운드리` → 반도체 노드 결과 → 클릭 시 그래프가 해당 노드로 이동 + 상세 패널
- [ ] `하림`(기업) → `하림 · 육계계열화업체` → 클릭 시 해당 노드로 이동
- [ ] 리서치 제목(예: `반도체`) → 리서치 결과 → 클릭 시 원문 글로 이동

## C. 노드 상세 + 다중 출처 근거 강도 (최종 #8, 성공기준 2)

- [ ] 노드 클릭 → 패널에 `기업/관계/근거` 카운트 + (여러 리서치가 받치면) `M개 리서치가 뒷받침` + `상세 보기`
- [ ] `상세 보기` → 노드 상세 페이지
- [ ] 헤더에 **`근거 N개 · 리서치 M개`** 강도 배지, 별칭 칩
- [ ] **근거 문단이 리서치 글별로 그룹**되어 표시(각 그룹에 `근거 K개`) → 출처 흐름 확인
- [ ] 여러 리서치가 받치는 노드 예: `육계계열화업체`(치킨산업+튀김유 → 리서치 2개)
- [ ] 근거 그룹의 리서치 제목 클릭 → 원문 글로 이동(문단 근거 확인)

## D. 모바일 (최종 #7, 성공기준 4)

- [ ] 개발자도구 반응형 ≈390px
- [ ] 지도 탐색/검색/필터 동작
- [ ] 노드 클릭 → 상세가 **하단 시트**로 표시 + `닫기`

## E. 관리자 인증 + 리서치 CRUD + 삭제 모달 (성공기준 1·3)

- [ ] `/admin` → 로그인 폼, 오답 실패 / 정답(`devgraph-admin`) → 리서치 글 목록
- [ ] 상단 네비: `리서치 글` · `지도 편집` · `감사 로그` · `공개 지도 ↗` · `로그아웃`
- [ ] `새 글` → 제목/본문(빈 줄로 문단 구분) → `초안 저장` → `draft` 뱃지
- [ ] `미리보기` 토글, `공개로`/`비공개로` 토글 → 공개 목록 노출/제거 확인
- [ ] `삭제` → **확인 모달**("리서치 글을 삭제할까요?…") → `취소`로 닫힘 / `삭제`로 삭제
- [ ] 삭제된 글에서만 추출됐던(고아) 노드가 공개 지도에서 사라짐(공유 노드는 유지)

## F. AI 추출 → 검수 → 승인 (최종 #4·#5, 성공기준 2·3)

관리자 → 글 편집 → **AI 추출** 패널.

- [ ] `AI 추출 실행`(실제 Codex) → ~90초 후 검수 화면 (키/로그인 없으면 명확한 에러 — 정상)
- [ ] 검수 3분할: **원문 근거 | 후보 그래프 | 후보 리스트**(노드/엣지/기업 역할/계층 관계/별칭/클러스터)
- [ ] 헤더에 **`model … · prompt … · 입력 글 …`** 실행 메타데이터 표시 (최종 #4)
- [ ] 노드 카드: 이름/타입/설명/근거문단 수정 → `저장`; `기존 노드 검색(병합)` → 결과 클릭 → "기존 노드 #N 와 병합" (최종 #5)
- [ ] 기업 카드: `기존 기업 검색(병합)` → 결과 클릭 → "기존 기업 #N 와 병합" (최종 #5, 예: `CJ`를 `CJ제일제당`으로 병합)
- [ ] 엣지 카드 `⇄ 방향 반전`, 각 그룹 `+ 추가`(수동 후보), 후보 `제외`/`포함`
- [ ] `포함 후보 일괄 승인` → 공개 지도 반영, 제외 후보는 미반영
- [ ] (병합 후) 노드 상세에서 기업 별칭이 보존됐는지 확인 가능

## G. 재추출 diff (성공기준 3)

- [ ] 이미 승인 그래프가 있는 글에서 다시 `AI 추출 실행`
- [ ] 후보에 **diff 뱃지** `추가`/`수정`/`삭제`
- [ ] 본문에서 사실을 지운 뒤 재추출 → 없어진 항목이 `삭제` 후보(읽기전용)로 표시
- [ ] `삭제` 포함 승인 → 이 글 근거 연결 제거 + 남은 근거 없으면 **비활성화**(공개 지도에서 사라짐); `제외` 시 미적용

## H. 관리자 노드 위치 편집 (최종 #2)

- [ ] `/admin/map`(지도 편집) → 드래그 가능한 그래프 + 미니맵
- [ ] 노드를 원하는 위치로 드래그 → `위치 저장` → `저장됨`
- [ ] 공개 지도(`/`) 새로고침 → 저장한 배치가 반영(좌표 없으면 자동 레이아웃)

- [ ] 액션 뱃지 `생성`/`수정`/`비활성화`/`삭제`, `원인`에 리서치 글 링크 또는 추출 run
